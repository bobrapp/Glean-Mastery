# Day 1 Lab
1. Pick one task you ask Glean to do the same way every week.
2. In Assistant, create a new Skill and set its scope to Personal (creator-only).
3. Write the instructions: what the Skill does, step by step.
4. Write the description — the "when to use" line that tells Glean which requests this Skill is for.
5. Run a request that matches the description and confirm your Skill handles it.

## Working Example: A Personal Skill's Shape

A Skill packages instructions Glean Assistant can reuse. A Personal Skill is
scoped to its creator — only you can invoke it. Verified against `assistant_skills`.

```text
Skill: "Weekly status digest"   (scope: Personal — only @you)
  Description (when to use):
    "When the user asks for a weekly status update or standup summary."
  Instructions:
    1. Pull my open Jira issues and recent PRs.
    2. Group them by project.
    3. Write 3 bullets: shipped, in progress, blocked.
```

The description is not decoration — it is what Glean reads to decide whether a
given request should trigger this Skill. A vague "when to use" line fires on the
wrong prompts; a specific one fires on exactly the requests you meant.
