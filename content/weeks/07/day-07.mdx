# Day 7 Lab
1. Survey the Web SDK surfaces: Chat, Autocomplete + Search, Modal, Sidebar, Recommendations.
2. Pick one for your app — inline Search box, a Modal launcher, or a docked Sidebar.
3. Mount the component into a container element with your backend-issued token.
4. Confirm that whichever surface you embed, results are still scoped to the signed-in user.

## Working Example: Embed the Chat Component

The Web SDK mounts a Glean surface into your page. You supply a container and an
auth token your backend minted for the signed-in user, and Glean handles
retrieval — still permission-scoped. Verified against `web_sdk`.

```typescript
import { GleanWebSDK } from "@gleanwork/web-sdk";

// Token is minted server-side for the signed-in user — never hard-coded.
const token = await fetchGleanTokenForCurrentUser();

GleanWebSDK.attach({
  instance: "acme",
  authToken: token,
  // Chat docked into a sidebar surface; swap `chat` for `search`,
  // `modal`, `sidebar`, or `recommendations` to change the surface.
  surface: "chat",
  target: document.getElementById("glean-chat")!,
});
```

Whichever surface you mount — Modal for an on-demand launcher, Sidebar for a
persistent panel — results stay scoped to the signed-in user. Verified against
`web_sdk`.
