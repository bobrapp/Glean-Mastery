# Day 1 Lab
1. Run a query for an active topic — something people edit often.
2. Add `updated:past_week` and note how the result count shrinks.
3. Swap in `updated:today`, then `updated:past_month`, and watch the window widen.
4. Pick the smallest window that still returns what you need.
5. Note when recency mattered more than another keyword would have.

## Working Example: updated: in Action

The `updated:` filter keeps only results changed inside a recency window. Verified
against `search_filters`.

```text
roadmap                       → everything ever indexed, oldest drafts included
roadmap updated:today         → only versions touched today
roadmap updated:past_week     → changes from the last seven days
roadmap updated:past_month    → a wider net for slower-moving work

Reach for updated: when the wrong results are stale, not off-topic.
```
