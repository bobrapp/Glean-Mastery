# Day 1 Lab
1. Open the Skills list and pick one built-in Skill that matches a task you do often.
2. Read its description — note the instructions and the tools it bundles.
3. Invoke it with one concrete input.
4. Read back the steps it ran and the sources it touched.
5. Note one task you repeat that a Skill could replace.

## Working Example: Invoking a Built-in Skill

A built-in Skill is a named package of instructions plus the tools to carry them
out — you invoke it instead of re-prompting the workflow each time. Verified
against `assistant_skills`.

```text
You:    @Skill "Meeting Prep" — prep me for the Acme renewal call

Skill bundles:
  • Instructions: gather context, summarize, list open risks
  • Tools:        enterprise search, calendar lookup, CRM read

Skill runs:
  1. Found the calendar event → attendees, time
  2. Searched: app:salesforce account:Acme  → open opportunity, stage
  3. Searched: Acme renewal                 → last QBR doc, 2 support tickets
  4. Returned: one-page brief + 3 open risks, each with a source link
```

The win is reuse: the instructions and the tool set travel with the Skill, so
the next person who invokes "Meeting Prep" gets the same workflow without
rebuilding the prompt. Verified against `assistant_skills`.
