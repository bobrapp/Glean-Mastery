# Day 2 Lab
1. Write one need as three keywords.
2. Write the same need as a full question.
3. Run both; compare what comes back — a list of docs vs a synthesized answer.
4. Note which mode fit the need better, and why.

## Working Example: Same Need, Two Modes

Glean accepts queries as keywords or as questions; the mode changes the shape of
the result. Verified against `search_basics`.

```text
Need: "What is our parental leave policy?"

Keyword mode:   parental leave policy
  → a ranked list of documents (you open the benefits doc yourself)

Question mode:  What is our parental leave policy?
  → a synthesized, cited answer drawn from the benefits docs

Rule of thumb:
  Know the exact document  → keywords (faster to the source)
  Want the answer itself   → a question (Glean assembles and cites it)
```
