# Day 1 Lab
1. Pick one project you work on.
2. List the entities around it: people, customers, related projects.
3. For each entity, name two "signals" — a doc, a ticket, a message, a channel.
4. Sketch how those connect. That shape is what the graph models for you.

## Working Example: An Entity and Its Signals

The Enterprise Graph models high-value entities and the network of signals around
each one. Verified against `enterprise_graph`.

```text
Entity: Project "Atlas"
  ├─ People:    owner @maya, contributors @raj @lin
  ├─ Customer:  Acme (linked opportunity)
  └─ Signals:
       • Design doc (Drive)        • 12 Jira issues (Sprint 7)
       • #atlas Slack channel      • 3 support tickets (Zendesk)
       • Launch deck (Drive)       • PR #482 (GitHub)
```

Search for "Atlas" and Glean can rank these together — not because the word
matches, but because the graph knows they belong to the same entity. That
relationship is the difference between enterprise search and a keyword box.
