# Day 1 Lab
1. Run a deliberately broad query — one that returns hundreds of mixed results.
2. Add one operator (`type:`, `app:`, `owner:`, or `after:`) and note the new count.
3. Add a second operator and note the count again.
4. Identify which single operator cut the most noise — that is your highest-value constraint.
5. Drop the weakest operator and confirm the result set barely changes.

## Working Example: Stacking Constraints in Sequence

Each operator is one constraint. Stack them one at a time and the result count
tells you which constraint actually mattered. Verified against `search_filters`.

```text
roadmap                                  → 520 results (docs, chats, tickets, decks)
roadmap type:document                    → 180 results (cut the chatter)
roadmap type:document app:gdrive         →  74 results (one source of truth)
roadmap type:document app:gdrive after:2026-01-01  → 19 results (this year only)

Read the deltas:
  520 → 180   type: did the heavy lifting (the wrong KIND was the noise)
   74 →  19   after: was the precision finisher
Start with the constraint that removes the wrong kind of thing first.
```
