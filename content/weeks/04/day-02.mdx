# Day 2 Lab
1. Run a broad query and open the facet panel beside the results.
2. Narrow once by clicking a datasource facet; note the result count.
3. Reproduce that same narrowing by typing the equivalent `app:` operator.
4. Decide which felt faster — and whether you'd want to repeat this search tomorrow.
5. Write the typed version down; that is the form you can save and share.

## Working Example: Same Narrowing, Two Doors

A facet click and a typed operator can land on the identical result set. The
difference is discovery versus repeatability. Verified against `datasource_filters`.

```text
Goal: just the Slack results for "incident postmortem"

  FACET path:    type the query → click the "Slack" datasource facet
                 (great when you don't yet know which apps have hits)

  OPERATOR path: type  incident postmortem app:slack
                 (great when you already know — and want to paste it again)

Rule of thumb:
  Exploring an unknown result set?      → click facets, let them show you
  Repeating a search you trust?         → type the operator, save the string
```
