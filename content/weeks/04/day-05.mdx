# Day 5 Lab
1. Choose one entity to investigate — a project, an incident, or a customer.
2. Run the topic query unscoped and note which apps surfaced hits.
3. Scope to each app in turn (`app:gdrive`, `app:slack`, `app:jira`) and read that system's slice.
4. Assemble the slices into a timeline: decision → discussion → execution.
5. Note any gap where one system has the story and another is silent — that gap is a finding.

## Working Example: One Topic, Several Systems

The graph relates an entity to signals scattered across apps. Scope to each app
to read its view, then stitch the views into one narrative. Verified against `enterprise_graph`.

```text
Entity: incident "INC-204"

  app:gdrive   incident INC-204   → the postmortem doc (the decisions)
  app:slack    incident INC-204   → the #incident channel (the live debate)
  app:jira     incident INC-204   → the remediation tickets (the follow-up work)

Stitch them in time order:
  Slack (what happened, live) → Jira (what we committed to fix)
                              → Drive (what we concluded, written up)
A silent system is itself a signal: no tickets means no follow-up was filed.
```
